# FitCheck! 개발 세션 인수인계서 (Session Handoff Notes)

본 문서는 다음 개발 세션의 AI 코딩 어시스턴트가 프로젝트의 맥락을 즉시 파악하고, 일관성 있는 설계를 이어갈 수 있도록 작성된 핏체크 V1 프로토타입 인수인계서입니다.

---

## 📂 1. 프로젝트 폴더 구조 및 설정 (Vite Base `/fitcheck/`)

프로젝트는 실서비스 서브디렉토리 배포 규칙에 맞추어 모든 소스 코드가 **`fitcheck/`** 폴더 내에 밀폐 구성되어 있습니다.

- **`fitcheck/index.html`**: 메인 마크업 (업로드, 로딩, 결과, 5:5 VS 매치, 모달 레이아웃 수록)
- **`fitcheck/style.css`**: 네오 브루탈리즘 스타일 및 펄스 애니메이션 수록
- **`fitcheck/app.js`**: 분석 로직, 배틀 라우팅, 모바일 포커싱 스크롤, 인스타 캔버스 익스포터 연동
- **`fitcheck/vite.config.js`**: `base: '/fitcheck/'`, `outDir: '../dist/fitcheck'` 배포 경로 바인딩
- **`fitcheck/assets/`**: OOTD 미리보기/예비용 에셋
- **`package.json` (Root)**: 빌드 명령어를 `fitcheck` 폴더 안으로 포워딩하는 프록시 스크립트 수록
  - `"dev": "npm run dev --prefix fitcheck"`
  - `"build": "npm install --prefix fitcheck && npm run build --prefix fitcheck"`
- **`backup/fitbattle/`**: 이전 작업물 원본 백업본 보관

## ⚡ 2. V1 프로토타입 핵심 구현 완료 사양

1. **배틀 매칭 시스템 (서버리스 수락 모드)**:
   - URL `?score=8450&tpo=일상` 파라미터 감입 시 자동으로 도전자 대결 모드로 스위칭.
   - 상황(TPO) 변경 방지 락(Lock) 처리, 배틀 수락 오렌지 배너 활성화, 버튼 🥊 갱신.
   - 로드 중 배틀 전용 멘트 순차 가동.

2. **5:5 VS 매칭 결과 판정 레이아웃**:
   - 결과 화면 이미지 박스 영역이 5:5 절반으로 쪼개져 상대방(👤 실루엣 더미 정보)과 나(도전자 OOTD)의 대결로 렌더링.
   - 득점에 따라 `WIN 🏆` 스탬프(민트색)와 `LOSE 💀` 스탬프(살몬색)가 각 플레이어 헤드 위로 동적 날인.
   - 카드 헤더의 우측 상태 배지가 `VICTORY 🎉` 또는 `DEFEAT 🚨`로 변형되어 전반적인 전투 판정 강조. (기존 이미지 가림 라벨은 배틀 모드 시 자동 숨김 처리됨)
   - 패배/승리에 따른 매운맛 비교 코멘트 출력.

3. **고도화된 모바일 핀(Pin) & 피드백 인터랙션**:
   - **차폐 예외화**: 피드백 팝업창 `#feedback-tooltip`을 이미지 내부 오버레이가 아닌 이미지 박스 아래의 블록형 서랍으로 재배치하여 OOTD의 모든 부위가 100% 가려짐 없이 보이게 수정.
   - **자동 스크롤 정렬**: 모바일 화면에서 팝업이 스크롤 아래로 접혀 보이지 않는 사용성 에러를 극복하기 위해, 핀 클릭 시 브라우저 레이아웃 대기 후 `scrollIntoView({ behavior: 'smooth', block: 'nearest' })`를 유려하게 호출.
   - **활성 하이라이팅**: 탭한 핀 마커에 강력한 검은색 링(`ring-[5px] ring-black`)과 `scale-110` 포커싱 적용.
   - **발견 유도 어포던스**: 😇/😈 마커에 입체적인 네오 쉐도우가 팽창/수축하는 펄스 애니메이션 탑재 및 하단 가이드 가시 팁 적용 (배틀 모드 시 가이드 팁은 자동 차단).

4. **코디 패치 및 가상 장착**:
   - 워스트 매치에서 "추천 코디 적용 ⚡" 시, 기존 😈 마커를 즉각 화면에서 감추어 복잡함을 제거.
   - 👟 독일군 스니커즈 스티커가 패치 지점에 스탬핑되며, 스티커를 누르면 토스트 가이드가 피드백 응답. 점수 +1,500점 카운팅 업 연출.

5. **인스타그램 스토리 Exporter**:
   - 인스타 공유 시 5:5 분할 OOTD, WIN/LOSE 도장 날인, 스코어 비교판이 결합된 1080x1920 세로형 규격 카드를 드로잉 후 저장.
   - 인스타그램 앱을 바로 호출할 수 있는 `instagram://` 가이드 딥링크 모달 연동.
   - 무신사 쇼핑몰 검색 이동 시, 안전 확인 과정을 거치는 컨펌 모달 `#musinsa-redirect-modal` 신설.

6. **다시 하기(Retry) URL 정화**:
   - 재시작 클릭 시 브라우저 주소창의 쿼리 파라미터를 말끔하게 청소(`replaceState`)하고 TPO 락 및 뱃지들을 원위치하여 즉시 일반 모드 진입이 가능하도록 제어.

## 🚀 3. 다음 세션 고도화 권장 로드맵 (AI API 연동 기획)

1. **AI API 연동 기준 세우기 (핵심 과제)**:
   - 다음 세션에서는 OOTD 이미지를 멀티모달 AI 모델(예: Gemini Pro Vision 또는 OpenAI GPT-4o) API에 전송하여 실질적인 코디 분석을 받아오도록 연동해야 합니다.
   - **API 연동 기획 기준**:
     - **인풋**: 사용자가 업로드한 OOTD 이미지 파일(Base64) + 선택한 TPO(일상, 데이트 등).
     - **아웃풋 JSON 스키마 합의 필요**: AI가 무작위 텍스트가 아닌 사전에 정의한 JSON 형식으로 일관되게 응답하도록 프롬프트를 설계해야 합니다.
       - `score`: 0 ~ 10,000 범위의 분석 점수.
       - `tier`: 패션 챌린저, 다이아몬드, 골드, 실버, 아이언 분기.
       - `roast`: 뼈 때리는 매운맛 한줄평 텍스트.
       - `bestMatch`: 가장 조화로운 부위 설명 및 베스트 좌표(x, y %).
       - `worstMatch`: 가장 아쉬운 부위 설명, 대체 아이템 추천명, 워스트 좌표(x, y %).
       - `musinsaQuery`: 추천 아이템 무신사 검색 키워드.
       - `stats`: TPO 스탯별 수치(0~100) 객체 배열.
2. **쇼핑 몰 다변화 및 동적 추천 데이터베이스 확장**:
   - AI 응답에서 뽑아낸 `musinsaQuery`를 통해 워스트 매치 툴팁의 Musinsa 이동 URL을 동적으로 주입합니다.
3. **Cloudflare Pages 배포 무결점 확인**:
   - 실서버 빌드 상태와 CORS 문제를 사전에 확인하여 API 호출이 프론트 단에서 원활하게 도는지 점검합니다.
